### Возможности

- **Современный интерфейс**: Чистый и интуитивный интерфейс на Vue 3
- **Многоязычная поддержка**: Упрощённый китайский, традиционный китайский (Тайвань/Гонконг), английский, японский, корейский и русский
- **Тёмная тема**: Автоматическое переключение тёмной/светлой темы по системным настройкам
- **Системная интеграция прокси**: Автоматическая настройка системного прокси для GNOME и KDE
- **Управление конфигурациями**: Лёгкое управление несколькими профилями SSR
- **Безопасность**: Надёжный бэкенд shadowsocksr-native

### Поддержка рабочих окружений

Приложение обеспечивает автоматическую настройку системного прокси для всех основных окружений Linux:

| Категория | Окружения | Метод прокси | Требуется перезапуск GUI |
|---|---|---|---|
| **Семейство GNOME** | GNOME, Ubuntu, Cinnamon, MATE, COSMIC, Budgie, Pantheon, Unity, Deepin, UOS, UKUI | gsettings | Нет |
| **KDE Plasma** | KDE Plasma 5/6, LXQt | kwriteconfig5/6 + dbus | Нет |
| **XFCE** | XFCE, Xubuntu | xfconf-query | Да |
| **Оконные менеджеры** | i3, sway, Hyprland, niri, Openbox, Fluxbox, dwm, awesome, bspwm, Qtile, herbstluftwm, icewm, jwm, xmonad, Enlightenment | Переменные окружения | Да |

**Терминальные приложения**: Во всех окружениях терминальные приложения автоматически используют SOCKS прокси после включения. Если окна терминала уже открыты, перезапустите их для применения настроек.

При использовании браузера Firefox необходимо отдельно настроить прокси в настройках браузера. Рекомендуется использовать расширение FoxyProxy для удобного переключения между прокси. Браузеры на базе Chromium автоматически используют системные настройки прокси (в GNOME/KDE).

### Поддержка архитектур

Проект поддерживает несколько архитектур CPU:
- **x86_64** (amd64) — предварительно собранные версии
- **i686** (32-бит x86) — сборка из исходников при необходимости
- **aarch64** (ARM 64-бит) — сборка из исходников при необходимости

**Примечание**: Предварительно собранные версии доступны только для архитектуры x86_64. Если вам нужны сборки для i686 или aarch64, скомпилируйте из исходников, следуя инструкциям ниже.

### Установка

#### Быстрая сборка (рекомендуется)

Используйте скрипт `build.sh` для автоматической сборки и упаковки:

```bash
git clone https://github.com/liangzhaoyuan12/shadowsocksr-client-linux.git
cd shadowsocksr-client-linux
npm install
./build.sh
```

В каталоге `build/` будут созданы:
- `*.deb` — пакет для Debian/Ubuntu
- `*.rpm` — пакет для Fedora/openSUSE
- `shadowsocksr-client-linux_{версия}_{архитектура}.tar.gz` — портативный архив (для пользователей не на deb/RPM)

#### Ручная сборка из исходников

1. Клонируйте репозиторий:
```bash
git clone https://github.com/liangzhaoyuan12/shadowsocksr-client-linux.git
cd shadowsocksr-client-linux
```

2. Установите зависимости:
```bash
npm install
```

3. Соберите приложение для вашей архитектуры:
```bash
npm run tauri build
```

Собранное приложение будет доступно в `src-tauri/target/release/bundle/`.

#### Кросс-компиляция для других архитектур

При необходимости сборки для другой архитектуры укажите целевую платформу:

**Для i686 (32-бит):**
```bash
# Установите целевую платформу
rustup target add i686-unknown-linux-gnu

# Сборка для i686
npm run tauri build -- --target i686-unknown-linux-gnu
```

**Для aarch64 (ARM 64-бит):**
```bash
# Установите целевую платформу
rustup target add aarch64-unknown-linux-gnu

# Сборка для aarch64
npm run tauri build -- --target aarch64-unknown-linux-gnu
```

**Примечание**: Кросс-компиляция может потребовать дополнительных системных зависимостей и инструментов. Убедитесь, что установлены соответствующие инструменты кросс-компиляции для целевой архитектуры.

#### Использование готового tar.gz пакета (для пользователей не на deb/RPM)

Для пользователей дистрибутивов, не использующих deb (Debian, Ubuntu и т.д.) или rpm (Fedora, openSUSE и т.д.), можно запустить приложение напрямую из архива `tar.gz`.

**Необходимые зависимости:**

Перед запуском приложения убедитесь, что установлены следующие системные библиотеки:

- **gtk3** — библиотека графического интерфейса GTK+
- **webkit2gtk4.1** — WebKitGTK для GTK 3 и libsoup 3

**Инструкция по установке:**

1. Скачайте пакет `tar.gz` со страницы релизов (например, `shadowsocksr-client-linux_0.3.0_x86_64.tar.gz`)
2. Распакуйте архив:
   ```bash
   tar -xzf shadowsocksr-client-linux_*.tar.gz
   ```
3. Перейдите в распакованную директорию и запустите исполняемый файл:
   ```bash
   cd shadowsocksr-client-linux
   ./shadowsocksr-client-linux
   ```

Если приложение не запускается, проверьте, установлены ли все необходимые зависимости.

### Использование

1. Запустите приложение из меню приложений или терминала
2. Нажмите «Добавить конфигурацию» для создания первого профиля SSR
3. Заполните данные сервера:
   - Имя профиля (только буквы)
   - Адрес сервера
   - Порт сервера
   - Пароль
   - Метод шифрования
   - Протокол
   - Обфускация
4. Выберите конфигурацию из списка
5. Нажмите «Включить прокси» для запуска прокси

#### Настройки локального прокси

После включения локальный SOCKS5 прокси будет доступен по адресу:
- **Адрес**: `127.0.0.1`
- **Порт**: Указанный в вашем профиле (по умолчанию: 1080)

#### Поведение прокси в разных окружениях

- **Семейство GNOME и KDE Plasma**: GUI-приложения автоматически применяют настройки прокси — перезапуск не требуется. Терминальные приложения будут использовать прокси после перезапуска окон терминала.
- **XFCE и оконные менеджеры (i3, sway, Hyprland и др.)**: Все приложения (GUI и терминал) необходимо перезапустить для применения настроек. Прокси настраивается через переменные окружения (`all_proxy`, `ALL_PROXY`), записываемые в `~/.config/environment.d/` и `~/.xprofile`.

### Технологический стек

- **Фронтенд**: Vue 3 с Composition API

- **Бэкенд**: shadowsocksr-native
- **Сборка**: Vite
