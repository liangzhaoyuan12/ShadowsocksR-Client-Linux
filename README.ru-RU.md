## Возможности

- **Умная маршрутизация**：Встроенный SOCKS5 прокси-роутер с 4 режимами маршрутизации — Обход LAN + Китай / Обход LAN / Обход Китая / Глобальный. Автоматически направляет внутренний трафик напрямую, а зарубежный — через прокси, исключая необходимость ручного переключения.
- **Двухпортовая архитектура**：Порт роутера (доступен для приложений, настраивается на панели управления) и сервисный порт SSR (внутренний, настраивается в редакторе) работают независимо. Системные настройки прокси (gsettings, kwriteconfig, переменные окружения) всегда указывают на порт роутера, обеспечивая согласованное поведение.
- **Управление переменными окружения**：Опциональный флажок — при включении системные переменные окружения (`all_proxy`/`ALL_PROXY`) управляются автоматически. При выключении используйте proxychains для маршрутизации отдельных команд без влияния на глобальное окружение.
- **Руководство по Proxychains**：Встроенные инструкции по настройке proxychains с динамическим отображением параметров.
- **Современный интерфейс**：Чистый и интуитивно понятный интерфейс на Vue 3 + Composition API.
- **Многоязычная поддержка**：7 языков — упрощённый китайский, традиционный китайский (Тайвань/Гонконг), английский, японский, корейский, русский.
- **Тёмная тема**：Автоматическое переключение между тёмной и светлой темой в зависимости от системных настроек.
- **Интеграция с системным прокси**：Автоматическая настройка для рабочих столов GNOME, KDE Plasma и XFCE.
- **Несколько конфигураций**：Удобное управление несколькими профилями SSR.

## Поддержка рабочих столов

| Категория | Рабочие окружения | Метод прокси | Перезапуск GUI |
|---|---|---|---|
| **Семейство GNOME** | GNOME, Ubuntu, Cinnamon, MATE, COSMIC, Budgie, Pantheon, Unity, Deepin, UOS, UKUI | gsettings | Нет |
| **KDE Plasma** | KDE Plasma 5/6, LXQt | kwriteconfig5/6 + dbus | Нет |
| **XFCE** | XFCE, Xubuntu | xfconf-query | Да |
| **Оконные менеджеры** | i3, sway, Hyprland, Openbox, Fluxbox, dwm, awesome, bspwm, Qtile, herbstluftwm, icewm, jwm, xmonad, Enlightenment | Переменные окружения (при включении) или proxychains | Да |

**Терминальные приложения**：При включённой опции «Изменять переменные окружения» новые терминалы автоматически используют SOCKS прокси. Существующие терминалы необходимо переоткрыть. На рабочих столах, отличных от GNOME/KDE, эта опция обязательна для поддержки прокси в терминале.

**Firefox**：Настройте прокси отдельно в параметрах браузера. Используйте расширение FoxyProxy для удобного переключения. Браузеры на базе Chromium автоматически следуют системным настройкам прокси (GNOME/KDE).

## Режимы умной маршрутизации

| Режим | Трафик LAN | Трафик Китая | Зарубежный трафик |
|---|---|---|---|
| **Обход LAN + Китай** (по умолчанию) | Напрямую | Напрямую | Через прокси |
| **Обход LAN** | Напрямую | Через прокси | Через прокси |
| **Обход Китая** | Через прокси | Напрямую | Через прокси |
| **Глобальный** | Через прокси | Через прокси | Через прокси |

Роутер работает как встроенный SOCKS5 сервер на настраиваемом порту роутера. Сопоставление IP-адресов Китая использует встроенные данные делегирования APNIC с бинарным поиском, обеспечивая производительность O(log n).

## Установка

### Быстрая сборка (рекомендуется)

```bash
git clone https://github.com/liangzhaoyuan12/shadowsocksr-client-linux.git
cd shadowsocksr-client-linux
npm install
./build.sh
```

Результат в `build/`：
- `*.deb` — Debian/Ubuntu
- `*.rpm` — Fedora/openSUSE
- `*.tar.gz` — Переносимый архив

### Ручная сборка

```bash
git clone https://github.com/liangzhaoyuan12/shadowsocksr-client-linux.git
cd shadowsocksr-client-linux
npm install
npm run tauri build
```

### Кросc-компиляция

```bash
# i686
rustup target add i686-unknown-linux-gnu
npm run tauri build -- --target i686-unknown-linux-gnu

# aarch64
rustup target add aarch64-unknown-linux-gnu
npm run tauri build -- --target aarch64-unknown-linux-gnu
```

### Использование переносимого tar.gz

Необходимые системные библиотеки：**gtk3**, **webkit2gtk4.1**

```bash
tar -xzf shadowsocksr-client-linux_*.tar.gz
cd shadowsocksr-client-linux
./shadowsocksr-client-linux
```

## Использование

1. Запустите приложение
2. Нажмите «**+ Добавить конфигурацию**» и заполните данные SSR сервера：
   - Имя профиля (только буквы)
   - Адрес и порт сервера
   - Пароль
   - Метод шифрования, протокол, обфускация
   - **Сервисный порт SSR** — внутренний порт для SSR sidecar (должен отличаться от порта роутера)
3. Выберите конфигурацию на боковой панели
4. На панели управления настройте **Порт роутера** (открытый SOCKS5 порт) — должен отличаться от порта SSR
5. Выберите **Режим маршрутизации** (рекомендуется «Обход LAN + Китай»)
6. При необходимости переключите «**Изменять переменные окружения**» (см. предупреждения под флажком)
7. Нажмите «**Включить прокси**»

### Настройки прокси

После включения приложения могут подключаться через：
- **SOCKS5**：`127.0.0.1:<router-port>` (отображается на панели управления)

### Без переменных окружения (Proxychains)

При выключенной опции «Изменять переменные окружения» используйте proxychains для маршрутизации отдельных команд：

```bash
# Установка
sudo apt install proxychains4

# Отредактируйте /etc/proxychains4.conf, добавьте в [ProxyList]：
socks5 127.0.0.1 1080

# Использование
proxychains curl https://www.google.com
proxychains git clone https://github.com/...
```

## Структура проекта

```
shadowsocksr-client-linux/
├── src/                    # Vue фронтенд
│   ├── components/         # Vue компоненты (ConfigForm, ConfigList, ProxyControl)
│   ├── locales/           # Файлы перевода на 7 языков
│   └── utils/             # Обёртки Tauri API
├── src-tauri/             # Tauri + Rust бэкенд
│   ├── binaries/          # Нативные бинарные файлы SSR sidecar (3 архитектуры)
│   ├── src/
│   │   ├── router.rs      # SOCKS5 прокси-роутер с умной маршрутизацией
│   │   ├── china_ip.rs    # Сопоставление IP-диапазонов Китая (данные APNIC)
│   │   ├── ssr_process.rs # Управление жизненным циклом sidecar
│   │   ├── proxy.rs       # Интеграция с системным прокси (GNOME/KDE/XFCE/WM)
│   │   └── model.rs       # Модели данных + перечисление RouteMode
│   └── tauri.conf.json
└── build.sh               # Скрипт сборки и упаковки в один клик
```
